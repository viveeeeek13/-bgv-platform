BGV Platform API

Complete REST API reference for the Background Verification Platform. Authenticate users, manage candidates, trigger Aadhaar & PAN verifications, and generate downloadable PDF reports.

REST / JSON JWT Auth Rate Limited v1.0.0 Node.js + Express MongoDB

✅

Authentication

All protected endpoints require a JWT Bearer token in the Authorization header.

Authorization Header
Authorization: Bearer <your_jwt_token>
ℹ️
Tokens expire after 7 days by default (configurable via JWT_EXPIRES_IN). After expiry, re-authenticate via /api/auth/login.

Error Handling

GET /api/candidates/:id 🔐 Auth Required Get candidate + verification logs
ℹ️
Aadhaar is returned masked as XXXX-XXXX-1234. Raw number is never exposed.
Response 200
{ "success": true, "data": { "candidate": { "_id": "65b1c2d3e4f5a6b7c8d9e0f1", "fullName": "John Doe", "email": "john.doe@example.com", "phone": "9876543210", "aadhaarNumber": "XXXX-XXXX-9012", "panNumber": "ABCDE1234F", "dob": "1995-06-15T00:00:00.000Z", "address": "123 MG Road, Bengaluru", "status": "VERIFIED", "maskedAadhaar": "XXXX-XXXX-9012", "verificationLogs": [ { "verificationType": "AADHAAR", "verificationStatus": "verified", "verifiedAt": "2026-05-21T09:00:12.000Z", "responsePayload": { "status": "verified", "nameMatch": true, "dobMatch": true } } ] } } }
PUT /api/candidates/:id 🔐 Auth Required Update candidate (PENDING status only)
⚠️
Editing is only allowed when candidate status is PENDING. Returns 400 after verification has started.
Same body fields as POST /api/candidates
200 OK
{ "success": true, "message": "Candidate updated.", "data": { "candidate": { ... } } }
400 Bad Request
{ "success": false, "message": "Cannot edit candidate after verification has started." }
DELETE /api/candidates/:id 🔐 Auth Required Delete candidate + all verification logs
🗑
This action is permanent and cascades — all associated VerificationLog records are also deleted.
200 OK
{ "success": true, "message": "Candidate deleted successfully." }

Verification Endpoints

POST /api/verifications/:id/start 🔐 Auth Required Run Aadhaar + PAN verification
ℹ️
No request body needed. The backend fetches candidate data and calls Aadhaar + PAN APIs automatically. Returns 400 if already VERIFIED.

Status Logic: Both pass → VERIFIED  |  Both fail → FAILED  |  One passes → PARTIAL

Response 200 — VERIFIED
{ "success": true, "message": "Verification complete. Status: VERIFIED", "data": { "overallStatus": "VERIFIED", "aadhaar": { "status": "verified", "result": { "status": "verified", "nameMatch": true, "dobMatch": true, "message": "Aadhaar verified successfully", "verifiedAt": "2026-05-21T09:00:12.000Z", "referenceId":"AADHAAR-1716286212000" } }, "pan": { "status": "verified", "result": { "status": "verified", "panStatus": "active", "message": "PAN verified successfully", "verifiedAt": "2026-05-21T09:00:13.600Z", "referenceId":"PAN-1716286213600" } } } }
Response 200 — FAILED
{ "success": true, "message": "Verification complete. Status: FAILED", "data": { "overallStatus": "FAILED", "aadhaar": { "status": "failed", "result": { "message": "Aadhaar verification failed: Record not found", "errorCode": "RECORD_NOT_FOUND" } }, "pan": { "status": "failed", "result": { "message": "PAN is inactive or deactivated", "errorCode": "PAN_INACTIVE" } } } }
GET /api/verifications/:id/logs 🔐 Auth Required Retrieve all verification history for a candidate
Response 200

Report Endpoints

GET /api/reports/:id 🔐 Auth Required Generate and download PDF report
⚠️
Returns 400 if candidate status is still PENDING. Run verification first.
📄
Response is a binary PDF stream with Content-Type: application/pdf and Content-Disposition: attachment; filename="BGV_Report_John_Doe_1716286400000.pdf"
Axios Example (Frontend)
// Download PDF in browser const response = await axios.get(`/api/reports/${candidateId}`, { headers: { Authorization: `Bearer ${token}` }, responseType: 'blob' }); const url = window.URL.createObjectURL(new Blob([response.data])); const link = document.createElement('a'); link.href = url; link.setAttribute('download', 'BGV_Report.pdf'); document.body.appendChild(link); link.click();
